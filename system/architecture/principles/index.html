    üöÄ 5. CI/CD: GitHub Actions¶

    ‚úÖ QA Deployment¶

    • Triggered on merge to main.
    • Auto deploys to OVH:
    • SSH
    • Rsync binaries
    • Generate .env
    • Restart systemd service

    ‚úÖ PROD Deployment¶

    • Manual via workflow_dispatch.

    üîê 6. Authentication with Auth0¶

    • Two tenants: qa-auth0, prod-auth0

    • Inject customer_id, org_id in access tokens
    • Used in both frontend and backend

    üìÑ 7. Logging Architecture¶

    • Serilog (JSON logs)
    • Logs written to:
    • /var/www/prod/logs/*.log
    • /var/www/qa/logs/*.log
    • Filename format: service-name-YYYYMMDD.log
    • Promtail regex extracts service/environment
    • Loki aggregates logs
    • Grafana dashboards include:
    • Error logs
    • Recent events
    • SQL queries via EF

    • Live stats
    • Alerts (future enhancement)
